Getting Started

Quick Start Guide

Get TweetPilot up and running in under 5 minutes. Follow these steps in order.

1
Download & Install
Visit the homepage and download the latest TweetPilot.dmg. Open the DMG, drag TweetPilot to your Applications folder, then launch it. macOS 12 Monterey or later required.
2
Install TweetClaw Extension
Install the TweetClaw Chrome extension. Then open x.com in Chrome — TweetClaw will auto-start a local WebSocket server on port 20086.
3
Configure an AI Model
In TweetPilot, go to Settings → AI Providers. Add your API key from OpenAI, Anthropic, or DeepSeek. This powers the AI chat and all automated actions.
4
Add Your Twitter Account
Go to Accounts in the sidebar and click Add Account. Select TweetClaw (browser mode) — TweetPilot will detect your logged-in session automatically.
5
Run Your First Script
Create a new workspace, open the AI chat, and try a command like "search tweets about AI automation from the last 24 hours". The agent will execute it using your connected account.
✓
All your data — credentials, scripts, workspaces — stays 100% local on your Mac. Nothing is uploaded to our servers except license validation.

Setup Steps

1
Open AI Provider Settings
In TweetPilot, click Settings in the top menu bar, then select AI Providers.
2
Choose a Provider
Select from the list: OpenAI, Anthropic, DeepSeek, Ollama, or Custom (any OpenAI-compatible endpoint). For Ollama, make sure the local server is running first.
3
Enter Your API Key
Paste your API key into the field. Keys are stored encrypted in your macOS Keychain — never sent to TweetPilot servers.
4
Select Model & Test
Pick a model from the dropdown, then click "Test Connection". A green checkmark confirms everything is working.
💡
For best results, use Claude Sonnet or GPT-4o — they have the largest context windows and best tool-calling accuracy for Twitter automation tasks.

Using Ollama (Local Models)

ℹ️
Ollama runs AI models entirely on your Mac. Install from ollama.ai, pull a model with ollama pull llama3, then select "Ollama" in TweetPilot settings. The default endpoint is http://localhost:11434.
Browser Extension

TweetClaw Extension

TweetClaw is a Chrome extension that acts as a bridge between TweetPilot and your Twitter/X browser session. No API keys needed — it uses your logged-in browser session directly.

✓
Using TweetClaw means no API rate limits, no developer account required, and access to all Twitter features that the official API doesn't expose.
1
Install from Chrome Web Store
Visit the TweetClaw Chrome Web Store page and click "Add to Chrome". Confirm the permissions prompt.
2
Open Twitter in Chrome
Navigate to x.com or twitter.com and make sure you are logged in. TweetClaw automatically starts a local WebSocket server on port 20086 when it detects a Twitter page.
3
Verify Connection in TweetPilot
In TweetPilot, go to Settings → TweetClaw. You should see a green "Connected" status. If not, try clicking "Reconnect".
4
Add Account via TweetClaw
Go to Accounts in TweetPilot, click Add Account, and select "TweetClaw (Browser Mode)". TweetPilot will detect which Twitter account is logged in and import it automatically.

Troubleshooting

⚠️
Port conflict: If port 20086 is in use by another app, change the port in TweetPilot settings under Settings → TweetClaw → Port, then restart Chrome.
⚠️
Not connecting: Try refreshing the Twitter tab in Chrome and clicking Reconnect in TweetPilot. Make sure Chrome is not blocking localhost connections.
Account Management

Managing Accounts

TweetPilot supports multiple Twitter/X accounts simultaneously. Each account runs in its own isolated workspace with dedicated credentials.

1
Open the Accounts Panel
Click "Accounts" in the left sidebar of TweetPilot. You'll see all connected accounts and their status.
2
Add a New Account
Click "Add Account" and choose a connection method: TweetClaw (browser session) for no API keys, or X API (direct) if you have developer credentials.
3
Switch Between Accounts
Use the account dropdown at the top of any workspace to switch context. Scripts and tasks in each workspace run under their assigned account.
4
Assign Accounts to Workspaces
Open a workspace's settings and set a "Default Account". All scripts and scheduled tasks in that workspace will use this account unless overridden.
💡
Use separate workspaces for different accounts to keep automation rules, Data Blocks, and schedules organized and isolated per account.
Automation

Task Scheduler

Run Python scripts automatically on a schedule. TweetPilot supports both interval-based timing and full cron expressions.

Pro Feature

Scheduled tasks require a Professional or Enterprise license. The free plan supports manual script execution only.

View Plans →
1
Write or Select a Script
Create a Python script in TweetPilot's script editor. The script should be self-contained — it runs without any user interaction.
2
Click "Schedule This Script"
In the script editor, click the clock icon or "Schedule" button in the toolbar. This opens the scheduling dialog.
3
Set Timing
Choose Interval (e.g., every 15 minutes) or Cron (e.g., 0 9 * * * for 9am daily). Cron uses standard 5-field syntax.
4
Enable & Monitor
Toggle the task on. Check the "Scheduled Tasks" panel to see upcoming runs, execution history, and output logs for each run.

Cron Expression Reference

Examples
0 9 * * * # Every day at 9:00 AM */15 * * * * # Every 15 minutes 0 9 * * 1-5 # Weekdays at 9:00 AM 0 */2 * * * # Every 2 hours 30 8,20 * * * # 8:30 AM and 8:30 PM daily
Analytics

Custom Reports

Build automated analytics reports with Python and send them to Feishu, email, or any webhook. Reports run on a schedule and deliver insights directly to your team.

1
Write a Report Script
Write a Python script that collects data (follower counts, engagement rates, tweet metrics) using the TweetPilot SDK, then formats it as text or markdown.
2
Configure Output Destination
Use the SDK's built-in output functions to send to Feishu (via webhook URL), print to the task log, or write to a local file for archiving.
3
Schedule It
Set a cron schedule (e.g., daily at 9am) so the report runs automatically. No manual triggering needed.

Example Report Script

daily_report.py
from tweetpilot import twitter, feishu # Fetch stats for active account stats = twitter.get_account_stats() report = f""" 📊 Daily Twitter Report Followers: {stats['followers']} (+{stats['new_followers']}) Tweets today: {stats['tweets_today']} Impressions: {stats['impressions']:,} Top tweet: {stats['top_tweet_url']} """ # Send to Feishu group feishu.send_message(report)
💡
Reports run as scheduled tasks — they require a Pro license. You can run them manually anytime from the script editor for testing.
API Integration

X API Configuration

Use the official Twitter/X API for enterprise-grade reliability. Required if you need v2 API endpoints or want to avoid browser-based automation entirely.

API v1.1
Full read/write access. Requires Basic developer plan or above.
API v2
Newer endpoints, timeline access. OAuth 2.0 with PKCE support.
1
Create a Developer Account
Go to developer.twitter.com and sign in with your Twitter account. Apply for developer access if you haven't already.
2
Create an App
In the Developer Portal, create a new App under a Project. Set the app permissions to "Read and Write" to enable posting and interactions.
3
Copy Your Keys
From the App's "Keys and Tokens" page, copy all 4 values: API Key, API Key Secret, Access Token, and Access Token Secret.
4
Paste into TweetPilot
In TweetPilot, go to Settings → X API. Paste all 4 keys and click Save. Then go to Accounts, add a new account, and choose "X API (Direct)".
⚠️
The free X API tier (formerly Essential) has strict rate limits. For automation-heavy workflows, we recommend upgrading to the Basic tier ($100/month) or using TweetClaw mode instead.
Notifications

Feishu / Lark Integration

Send automated reports, alerts, and summaries directly to your Feishu (Lark) group chats. Set it up in minutes with a webhook URL.

1
Create a Bot in Your Group Chat
In Feishu/Lark, open the group chat where you want to receive reports. Click the "..." menu → "Bots" → "Add Bot" → "Custom Bot".
2
Copy the Webhook URL
After creating the bot, copy the webhook URL provided. It looks like: https://open.feishu.cn/open-apis/bot/v2/hook/xxxxx
3
Add to TweetPilot
In TweetPilot, go to Settings → Feishu. Paste the webhook URL and click "Test" to send a test message. You should see it appear in your group chat immediately.
4
Use in Scripts
In your Python scripts, call feishu.send_message("your text") to push messages to the configured group. Supports plain text and markdown.
